# CIO 投资操作系统 —— 全天候（All Weather）基金构建

这是一个**给自己使用的长期文档仓库**，用于系统性构建一套  
👉 可解释  
👉 可执行  
👉 可复盘  
👉 可迭代  

的 **全天候（All Weather）投资组合与研究体系**。

核心目标只有一句话：

> 把宏观叙事写成规则，把规则落实为仓位，把仓位纳入风控，并形成持续进化的投资操作系统。

---

## 这个仓库解决什么问题

- 不再停留在“宏观观点 / 市场感觉”
- 将 **增长 / 通胀 / 政策 / 信用** 明确为可观测、可切换的状态
- 构建一个在不同“宏观天气”下都**尽量不被单一情景击穿**的组合骨架
- 把投资行为变成：
  - 有假设
  - 有证伪
  - 有记录
  - 能复盘
  - 能改进

这是一个**工作系统**，不是投资笔记。

## 文档写作与结构约定（强烈建议遵守）

为了保证**长期可维护性**，每一份“规则 / 策略 / 模块”文档，尽量包含以下结构：

- **目标**：这个模块解决什么问题  
- **核心假设**：为什么它应该有效  
- **作用机制**：通过哪条路径影响收益或风险  
- **证伪条件**：什么情况下我承认它是错的  
- **执行规则**：如何转化为仓位 / 参数 / 行动  
- **风险控制**：最坏情况如何不被打穿  
- **复盘频率**：多久检查一次是否仍然成立  
- **变更记录**（可选）：重要修改与原因

---

## 决策纪律（这是系统能否进化的关键）

任何**主观判断 / 临时调整 / override**，都必须留下记录，包括：

- 触发原因（数据 / 事件 / 判断）
- 预期作用路径（贴现率 / 盈利 / 风险溢价）
- 明确的失效条件（什么时候算判断错误）
- 复查日期（到期必须复盘）
- 回滚方案（如何撤回这次调整）

> 不记录的判断，等于没有发生过。
